# FinTechTerms

**Трехъязычная платформа финансовой, финтех- и IT-терминологии для турецких студентов, изучающих русский язык.**

FinTechTerms - это цифровой образовательный проект для конкурса
**«Дипломаты русского слова» / «Rusca Sozunun Diplomatlari»**. Платформа
продвигает русский язык через практическую профессиональную лексику: финансы,
финтех, цифровой банкинг, платежи, криптовалюты, кибербезопасность и
информационные технологии.

## Почему проект соответствует конкурсу

Конкурс ориентирован на проекты, которые продвигают русский язык в Турции,
помогают студентам, интересующимся русским образованием, и адаптированы к
турецкой образовательной и медийной среде. FinTechTerms соответствует этим
критериям напрямую:

- Проект показывает русский язык как практический инструмент для современных
  финансовых и технологических сфер.
- Турецкий студент получает связь между русской, турецкой и английской
  терминологией.
- Платформа уже доступна онлайн и может использоваться студентами,
  преподавателями и самостоятельными учащимися.
- Избранное, тесты и интервальное повторение превращают пассивное чтение в
  активную учебную практику.
- Telegram-бот учитывает цифровые привычки пользователей в России и странах
  СНГ: термин дня, мини-тесты и произношение доступны в формате мессенджера.
- Русскоязычная терминологическая база может расширяться после конкурса и
  работать как долгосрочный образовательный ресурс.

## Короткая презентация

Турецкий студент, изучающий русский язык и интересующийся экономикой,
финансовыми технологиями, цифровым банкингом или IT, часто сталкивается с
проблемой: обычные словари дают перевод, но не объясняют профессиональный
контекст термина.

FinTechTerms решает эту проблему. Пользователь открывает русский финтех-термин,
видит турецкий и английский эквиваленты, читает объяснение, сохраняет термин в
избранное и возвращается к нему через тесты и интервальное повторение.

## Видение разработчика и исходная идея проекта

Как студент, обучающийся по двум направлениям - управленческие информационные
системы и экономика, я связываю свои академические и профессиональные цели с
продолжением образования на уровне магистратуры в ведущих университетах
России. В процессе подготовки я понял, что профессионально следить за
современными финансами, информационными системами и технологиями на русском
языке невозможно только через стандартное языковое обучение. Для этого нужна
гораздо более специфическая профессиональная терминология, особенно на уровне
B2+ и выше.

FinTechTerms был создан не только как личный инструмент обучения. Это
устойчивый цифровой мост для турецких студентов, которые разделяют похожие
карьерные цели и хотят использовать академические и профессиональные
возможности, связанные с Россией.

## Слой Telegram-бота

Telegram-бот показывает, что FinTechTerms не ограничивается веб-сайтом:
проект переносит обучение в мессенджерный формат, который особенно привычен
для цифровой среды России и стран СНГ. Для жюри это важно, потому что проект
учитывает реальные пользовательские привычки целевой аудитории.

Слой бота поддерживает:

- регулярное микрообучение через термин дня;
- поиск по общей базе из 956 терминов прямо внутри Telegram;
- мини-тесты для активного вспоминания;
- голосовое произношение терминов на русском, турецком и английском языках;
- статистику прогресса, которая делает учебное поведение видимым.

Такой подход превращает FinTechTerms из статического словаря в многоканальную
образовательную экосистему на базе web + Telegram.

## Образовательная ценность

FinTechTerms построен вокруг прикладного изучения языка:

- **Контекст прежде всего:** каждый термин связан с реальным финансовым или
  техническим понятием.
- **Многоязычный перенос:** турецкий студент сравнивает русский с турецким и
  английским языками.
- **Профессиональная релевантность:** лексика полезна для учебы, стажировок,
  деловой коммуникации и технологической карьеры.
- **Закрепление знаний:** тесты и интервальное повторение переводят чтение в
  активное вспоминание.
- **Доверие:** страницы источников, методологии и редакционной политики
  уменьшают риск поверхностной или ошибочной терминологии.

## Вклад в продвижение русского языка

Проект показывает русский язык как рабочий язык современной финансовой и
технологической информации. Русский интерфейс не является символическим
переводом: в проекте есть русские словарные страницы, русские метаданные,
русская тематическая навигация и русские объяснения, связанные с
профессиональной терминологией.

Это делает платформу полезной для турецких студентов, которые рассматривают
русскоязычные образовательные программы, российские университеты или
профессиональную коммуникацию на русском языке.

## Архитектура

FinTechTerms - это готовое к рабочей эксплуатации приложение на Next.js и
Supabase с отдельной Python-средой выполнения для Telegram-бота. Бот переносит
ту же терминологическую базу в привычный для русскоязычной цифровой среды
мессенджерный формат: термин дня, поиск, мини-тесты, голосовое произношение и
статистика прогресса.

```text
FinTechTerms Website/
├── app/                  # Веб-приложение Next.js App Router
├── components/           # React-компоненты
├── contexts/             # Провайдеры Auth, SRS, Language, Theme, Toast
├── lib/                  # Supabase, валидаторы, AI, rate limiting, SEO
├── utils/                # Алгоритм SRS, хранилище, вспомогательные функции
├── data/                 # Статические данные терминов и SEO
├── locales/              # Переводы en, tr, ru
├── __tests__/            # Unit- и integration-тесты Jest
├── e2e/                  # E2E-тесты Playwright
├── docs/                 # Документация по архитектуре, безопасности и эксплуатации
├── public/               # PWA-ресурсы и статические файлы
├── telegram-bot/         # Python-среда Telegram-бота
└── .github/workflows/    # CI/CD и выпускные проверки
```

## Быстрый старт для разработчика

### Веб-приложение

```bash
npm install
cp .env.example .env.local
npm run dev
```

После клонирования нужно один раз запустить `npm run dev` перед открытием
проекта в IDE. Next.js сгенерирует `next-env.d.ts`, который нужен локальным
TypeScript-инструментам.

Каноническая схема базы данных находится в `supabase/migrations/`, начиная с
`20260306000000_canonical_baseline.sql`. Изменения схемы должны проходить
только через цепочку миграций. Архивные SQL-файлы из `lib/` не нужно выполнять
вручную.

### Telegram Bot

```bash
cd telegram-bot
python -m venv venv
source venv/bin/activate
pip install -r requirements.txt
cp .env.example .env
python -m bot
```

Корневой [`.env.example`](.env.example) описывает переменные web app и общих
скриптов. Бот использует `telegram-bot/.env`.
