# FinTechTerms

**Türkiye'deki Rusça öğrenenler için üç dilli fintech, finans ve IT terminoloji platformu.**

FinTechTerms, **Rusça Sözünün Diplomatları / Дипломаты русского слова** proje
yarışması için hazırlanmış, canlı çalışan bir dijital eğitim ürünüdür. Proje
Rusça'yı genel kelime ezberiyle değil; finans, fintech, dijital bankacılık,
ödemeler, kripto, siber güvenlik ve IT gibi mesleki alanlar üzerinden öğretir.

## Kısa Rusça Özet

FinTechTerms - это трехъязычная образовательная платформа для турецких
студентов, изучающих русский язык через лексику финансов, финтеха и
информационных технологий. Проект помогает сделать русский язык практическим
инструментом для учебы, стажировок и профессионального общения.

## Yarışmaya Uygunluk

Yarışma, Türkiye'de Rus dilinin ve Rusça eğitiminin yaygınlaştırılmasına hizmet
eden, öğrencilere somut fayda üreten ve Türkiye'nin eğitim-medya ortamında
kullanılabilecek projeleri arıyor. FinTechTerms bu ihtiyaca doğrudan cevap
verir:

- Rusça'yı yalnızca genel kültür veya temel kelime bilgisiyle sınırlamaz;
  finans ve teknoloji gibi kariyer değeri yüksek alanlara taşır.
- Türk öğrenciler için Rusça, Türkçe ve İngilizce arasında kavramsal köprü
  kurar.
- Jüri, öğretmenler ve öğrenciler tarafından hemen açılabilecek canlı bir
  dijital üründür.
- Quiz, favoriler, tekrar akışları ve konu merkezleriyle pasif okumayı aktif
  öğrenmeye çevirir.
- Telegram botu sayesinde Rusya ve BDT dijital alışkanlıklarına yakın bir
  mesajlaşma kanalında günlük terim, mini quiz ve telaffuz desteği sunar.
- Yarışmadan sonra da genişleyebilecek kamusal bir Rusça terminoloji yüzeyi
  oluşturur.

## Kısa Sunum Metni

Rusça öğrenen bir Türk öğrenci ekonomi, finansal teknolojiler, dijital
bankacılık veya IT alanında çalışmak istediğinde klasik sözlükler genellikle
yalnızca çeviri verir. Fakat mesleki hayatta gereken şey sadece kelimenin
karşılığı değil; kavram, kullanım bağlamı, risk, alan ilişkisi ve doğru
terminolojidir.

FinTechTerms bu boşluğu kapatır. Kullanıcı bir Rusça fintech terimini açabilir,
Türkçe ve İngilizce karşılıklarıyla karşılaştırabilir, kaynak destekli
açıklamasını okuyabilir, favorilerine ekleyebilir ve quiz/aralıklı tekrar
akışıyla yeniden çalışabilir.

## Geliştiricinin Vizyonu ve Çıkış Noktası

Yönetim Bilişim Sistemleri ve Ekonomi alanlarında çift anadal yapan bir öğrenci
olarak akademik ve profesyonel hedefim, eğitimimi Rusya'daki önde gelen
üniversitelerde yüksek lisans seviyesine taşımaktır. Bu hazırlık sürecinde,
modern finans, bilişim ve teknoloji dünyasını Rusça üzerinden profesyonel
düzeyde takip etmenin standart dil eğitiminden çok daha spesifik bir mesleki
terminoloji, özellikle B2+ seviyesinde alan dili, gerektirdiğini fark ettim.

FinTechTerms yalnızca kişisel bir öğrenme aracı olarak değil; benimle aynı
kariyer hedeflerini paylaşan, Rusya'daki akademik ve profesyonel fırsatları
değerlendirmek isteyen Türk gençleri için sürdürülebilir bir dijital köprü
olarak inşa edilmiştir.

## Telegram Bot Katmanı

Telegram botu, FinTechTerms'in yalnızca web sitesi olmadığını; Rusya ve BDT
ekosisteminde güçlü karşılığı olan bir mesajlaşma kanalına da taşındığını
gösterir. Bu katman jüri açısından önemlidir çünkü proje, hedef kitlenin gerçek
dijital alışkanlıklarına yaklaşır.

Bot katmanı şu işlevleri destekler:

- Günlük terim akışıyla düzenli mikro-öğrenme.
- 956 terimlik ortak veritabanı üzerinde Telegram içinden arama.
- Mini quiz akışıyla aktif hatırlama pratiği.
- Rusça, Türkçe ve İngilizce terimler için sesli telaffuz desteği.
- Öğrenme davranışını görünür kılan ilerleme istatistikleri.

Bu yapı, FinTechTerms'i yalnızca statik bir sözlükten çıkarıp web + Telegram
tabanlı çok kanallı bir öğrenme ekosistemine dönüştürür.

## Eğitim Değeri

FinTechTerms uygulamalı dil edinimi için tasarlandı:

- **Bağlam önce gelir:** her terim gerçek bir finansal veya teknik kavram
  içinde açıklanır.
- **Çok dilli transfer:** Türk öğrenci Rusça'yı Türkçe ve İngilizceyle
  karşılaştırarak öğrenir.
- **Mesleki ilişki:** terimler eğitim, staj, iş görüşmesi, akademik çalışma ve
  teknoloji kariyeri için kullanılabilir.
- **Kalıcılık:** quiz ve aralıklı tekrar akışları pasif okumayı aktif hatırlama
  pratiğine dönüştürür.
- **Güven:** kaynak, metodoloji ve editoryal politika sayfaları yüzeysel veya
  hatalı terminoloji riskini azaltır.

## English Technical Snapshot

FinTechTerms is a production-grade Next.js and Supabase application with a
separate Python Telegram bot runtime. The web app uses the Next.js App Router,
TypeScript, Tailwind CSS, Supabase Auth/RLS, OpenRouter-backed AI study routes,
Jest, Playwright, GitHub Actions, and Vercel. The Telegram bot extends the same
term base into a messaging workflow with daily terms, search, mini quizzes,
voice pronunciation, and progress statistics.

```text
FinTechTerms Website/
├── app/                  # Next.js App Router web app
├── components/           # React components
├── contexts/             # Auth, SRS, Language, Theme, Toast providers
├── lib/                  # Supabase, validators, AI, rate limiting, SEO
├── utils/                # SRS algorithm, storage, helpers
├── data/                 # Static term and SEO data
├── locales/              # en, tr, ru translations
├── __tests__/            # Jest unit and integration tests
├── e2e/                  # Playwright E2E tests
├── docs/                 # Architecture, security, operations documentation
├── public/               # PWA assets and static files
├── telegram-bot/         # Python Telegram bot runtime
└── .github/workflows/    # CI/CD and preview release gates
```

### Developer Quick Start

```bash
npm install
cp .env.example .env.local
npm run dev
```

After cloning, run `npm run dev` once before opening the project in an IDE.
Next.js generates `next-env.d.ts` on first run, and TypeScript tooling expects
that file locally.

The canonical database schema lives in `supabase/migrations/`, starting with
`20260306000000_canonical_baseline.sql`. Apply schema changes through the
migration chain only. Do not manually execute archived SQL files from `lib/`.

### Telegram Bot

```bash
cd telegram-bot
python -m venv venv
source venv/bin/activate
pip install -r requirements.txt
cp .env.example .env
python -m bot
```

The root [`.env.example`](.env.example) is the catalog for web app and shared
script variables. The bot uses `telegram-bot/.env`.

### Verification Commands

```bash
npm run typecheck
npm run lint
npm test -- --runInBand
npm run verify:bootstrap-db
npm run verify:release-db
npm run test:e2e:guest
npm run test:e2e:auth
npm run smoke:staging
cd telegram-bot && python -m pytest tests/ -v
```

Preview release verification runs automatically on pushes and pull requests to
`main` through `.github/workflows/e2e.yml`.
